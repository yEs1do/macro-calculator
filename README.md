# 🎯 宏观阵型 Delta 计算器 (Macro Formation Delta Calculator)

> **“在量化执行阶段，精确到每一元的调仓是抗衡交易情绪的最后一道防线。”**

本仓库是 [🌌 宏观动量双轨制量化实盘系统](https://github.com/yEs1do/Nasdaq_per_Oz) 的官方配套执行终端。它是一个纯静态、高性能的调仓计算工具，旨在解决量化信号产生后，如何将“比例建议”快速转化为“挂单金额”的实战痛点。

---

## 🛠️ 核心功能 (Core Features)

* **双轨制无缝兼容**：支持“纯宏观底座”与“70/30 宏观+动量”双模式。动量金额输入 0 即为纯宏观，输入 >0 则自动激活双轨全局计算。
* **智能跨策略收割 (5% 宽容度)**：内置 25%~35% 的动量占比红线。大赚时强制收割利润反哺宏观，暴跌时启动“剥洋葱法”抽血债基救济动量。
* **Delta 差额算法**：输入当前各资产实盘余额，自动计算与目标阵型之间的“Delta（差额）”，直接给出优先级排序的（卖出 ➡️ 买入 ➡️ 稳住）指令单。
* **动态金价锚定**：通过 URL 参数无缝接收来自 Python 后端的实时人民币金价数据。
* **动态调仓阈值**：支持自定义忽略微小净值漂移的金额阈值（如 100 元），避免过度交易产生的无意义滑点。
* **本地沙盒记忆**：利用浏览器 `LocalStorage` 存储实物金克数与调仓阈值，确保“一次输入，永久记忆”且零隐私泄露。

## 🏗️ 阵型与调仓逻辑 (Formation Logic)

计算器严格遵循双轨制系统的底层逻辑，分为【全局跨策略平衡】与【宏观底座内平衡】两个层级：

### ⚔️ 全局控制：动量策略 5% 宽容度法则
如果填入了动量策略资金（基准目标为总资产的 30%），系统将优先进行以下判定：
1. **⚪ 稳住不动 (25% ~ 35%)**：动量处于宽容度内，系统不予干涉，让利润继续奔跑。
2. **🔴 大赚收割 (> 35%)**：动量在牛市暴涨，系统强行卖出超标利润，转入宏观底座买入避险资产，锁定胜局。
3. **🟢 剥洋葱法补血 (< 25%)**：动量暴跌，系统强行启动救济。严格按 **【1.现金短债 ➡️ 2.红利长债】** 的单向顺序抽血，去补齐动量缺口。救济期间**绝对锁定纳指与黄金**，不允许卖出核心资产！

### 🛡️ 宏观底座 (70% 仓位)：双模作战阵型
在处理完动量资金后，剩余的宏观资金将根据 MA200 均线信号，严格按以下阵型分配（若单项资产的漂移金额 ≤ 设定的阈值，则提示无需操作）：

**🟩 进攻期 (Attack Mode：Nasdaq_per_Oz MA10 > MA200)**
1. **现金/短债**: 10% (机动部队)
2. **红利/长债**: 10% (防御补充)
3. **纳指 ETF**: 60% (核心主力)
4. **黄金资产**: 20% (侧翼掩护)

**🟥 防御期 (Defense Mode：Nasdaq_per_Oz MA10 < MA200)**
1. **现金/短债**: 30% (流动性堡垒)
2. **红利/长债**: 20% (抗通胀防线)
3. **纳指 ETF**: 10% (保留观察仓)
4. **黄金资产**: 40% (核心避险)

## 🚀 部署与使用 (Deployment)

1.  **托管**：本项目建议托管于 **GitHub Pages**。
2.  **调用**：由主仓库脚本自动生成带金价参数的链接：
    `https://yEs1do.github.io/macro-calculator/?gold=XXXX.XX`
3.  **执行流程**：
    * 点击钉钉推送链接。
    * 填入前 4 项宏观资产市值，以及第 5 项实物金克数。
    * **填写第 6 项动量金额**（若当前不含动量体系，直接保持默认值 0 即可）。
    * 确认底部的【调仓忽略阈值】（建议 100 元）。
    * 根据系统生成的红色（卖出）和绿色（买入）清单执行挂单。**操作准则：先执行卖出以回笼资金，差额不足使用活期垫资款抢跑。**
